=== ОБЪЯСНЕНИЕ ПЕРЕМЕННЫХ ETEXT, EDATA, END ===

Программа process_memory.c демонстрирует организацию виртуальной памяти процесса
в операционных системах Unix/Linux. Переменные etext, edata и end являются
специальными символами, которые определяются компоновщиком (linker) и указывают
на границы различных сегментов памяти.

1. ETEXT (End of TEXT segment)
   - Адрес первого байта ПОСЛЕ текстового сегмента
   - Текстовый сегмент содержит:
     * Исполняемый код программы (инструкции)
     * Код всех функций (main, showit и др.)
   - Особенности:
     * Только для чтения (read-only)
     * Защищен от модификации
     * Разделяется между всеми запущенными копиями программы
   - Располагается в самой нижней части виртуальной памяти процесса

2. EDATA (End of DATA segment) 
   - Адрес первого байта ПОСЛЕ сегмента инициализированных данных
   - Сегмент данных содержит:
     * Глобальные переменные с явной инициализацией
     * Статические переменные с явной инициализацией
     * Строковые константы
   - Пример из программы:
     char *cptr = "This message is output by the function showit()\n";
   - Особенности:
     * Имеет начальные значения
     * Доступен для чтения и записи

3. END (End of BSS segment)
   - Адрес первого байта ПОСЛЕ сегмента BSS (Block Started by Symbol)
   - BSS сегмент содержит:
     * Глобальные переменные БЕЗ явной инициализации
     * Статические переменные БЕЗ явной инициализации
   - Пример из программы:
     char buffer1[25]; (без инициализации)
   - Особенности:
     * При запуске программы заполняется нулями
     * Экономит место в исполняемом файле
     * Доступен для чтения и записи

=== СТРУКТУРА ВИРТУАЛЬНОЙ ПАМЯТИ ПРОЦЕССА ===

Низкие адреса
│
├── Текстовый сегмент (Code)
│   ├── Исполняемые инструкции
│   ├── Функции программы
│   └── [&etext] ← КОНЕЦ ТЕКСТОВОГО СЕГМЕНТА
│
├── Сегмент данных (Initialized Data)
│   ├── Инициализированные глобальные переменные
│   ├── Статические переменные с значениями
│   └── [&edata] ← КОНЕЦ СЕГМЕНТА ДАННЫХ
│
├── BSS сегмент (Uninitialized Data)
│   ├── Неинициализированные глобальные переменные
│   ├── Статические переменные без значений
│   └── [&end] ← КОНЕЦ BSS СЕГМЕНТА
│
├── Куча (Heap)
│   ├── Динамическая память (malloc, calloc, realloc)
│   ├── Растет в сторону увеличения адресов ↑
│   └── Пример: buffer2 в функции showit()
│
├── ... Свободная память ...
│
└── Стек (Stack)
    ├── Локальные переменные функций
    ├── Аргументы функций
    ├── Возвращаемые адреса
    ├── Растет в сторону уменьшения адресов ↓
    └── Пример: переменная i в main(), buffer2 в showit()

Высокие адреса

=== ПРАКТИЧЕСКОЕ ЗНАЧЕНИЕ ===

Понимание расположения переменных в памяти важно для:

1. ОПТИМИЗАЦИИ ПАМЯТИ
   - Локальные переменные в стеке быстро доступны
   - Глобальные переменные сохраняются все время работы программы

2. ОТЛАДКИ ПРОГРАММ
   - Анализ segmentation fault
   - Понимание где находится ошибочный указатель

3. БЕЗОПАСНОСТИ
   - Переполнение буфера в стеке может перезаписать возвращаемые адреса
   - Переполнение кучи может повредить структуры управления памятью

4. РАЗРАБОТКИ СИСТЕМНОГО ПО
   - Создание загрузчиков
   - Реализация механизмов управления памятью

=== ПРИМЕР ИЗ ПРОГРАММЫ ===

В программе process_memory.c:

- main(), showit()    → между началом и &etext (текстовый сегмент)
- cptr                → между &etext и &edata (сегмент данных)
- buffer1             → между &edata и &end (BSS сегмент)
- i в main()          → в стеке (высокие адреса)
- buffer2 в showit()  → в куче (между &end и стеком)

Эти переменные помогают понять организацию памяти и используются в системном
программировании для анализа структуры выполняющихся процессов.
